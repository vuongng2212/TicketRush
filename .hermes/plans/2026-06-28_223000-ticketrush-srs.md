# Tài Liệu Nghiệp Vụ & Kỹ Thuật (SRS & SDS) - TicketRush

Hệ thống bán vé concert chịu tải cao (High-Throughput Concert E-Ticketing System) tích hợp GraphQL và gRPC.

---

## 1. MỤC TIÊU DỰ ÁN (Project Goal)
Xây dựng một hệ thống bán vé Concert (Monolith Backend bằng Spring Boot 3.3, Frontend bằng Next.js 14+) mô phỏng lại một sự kiện mở bán vé lớn (Flash Sale).
Hệ thống giải quyết các bài toán cốt lõi:
- **Tải cao (High Concurrency):** Ngăn chặn quá tải Database (PostgreSQL) khi lượng truy cập đặt vé đạt tới 10,000+ RPS bằng cơ chế lọc trên Redis.
- **Tính nhất quán (Consistency):** Đảm bảo 0% bán trùng ghế (Double Booking) và giải phóng ghế thông minh (Ticket Expiration/Timeout) qua RabbitMQ.
- **Trải nghiệm giao diện (Rich UX):** Hiển thị sơ đồ ghế động SVG, đồng bộ hóa trạng thái ghế real-time cho hàng ngàn người dùng cùng lúc qua GraphQL Subscriptions (WebSockets).
- **Giao tiếp B2B chuẩn gRPC:** Xây dựng ứng dụng giả lập quét mã vé check-in tại cổng sự kiện thông qua gRPC Server hiệu năng cao.

## 3. THIẾT KẾ DATABASE SCHEMA (PostgreSQL)

```sql
-- Schema: ticket_rush

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE concerts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(255) NOT NULL,
    description TEXT,
    start_time TIMESTAMP NOT NULL,
    venue VARCHAR(255) NOT NULL,
    status VARCHAR(50) DEFAULT 'UPCOMING', -- UPCOMING, OPEN, CLOSED
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE seat_zones (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    concert_id UUID REFERENCES concerts(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL, -- VIP, CAT1, CAT2, Standard
    price DECIMAL(12, 2) NOT NULL,
    capacity INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE seats (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    zone_id UUID REFERENCES seat_zones(id) ON DELETE CASCADE,
    seat_number VARCHAR(20) NOT NULL, -- A1, A2, B1, etc.
    status VARCHAR(50) DEFAULT 'AVAILABLE', -- AVAILABLE, HELD, SOLD
    locked_by_user_id UUID REFERENCES users(id),
    locked_until TIMESTAMP,
    version INT DEFAULT 0, -- Dùng cho Optimistic Locking ở DB (nếu cần)
    CONSTRAINT unique_zone_seat UNIQUE(zone_id, seat_number)
);

CREATE TABLE orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    concert_id UUID REFERENCES concerts(id),
    status VARCHAR(50) DEFAULT 'PENDING', -- PENDING, COMPLETED, EXPIRED, CANCELLED
    total_amount DECIMAL(12, 2) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE tickets (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_id UUID REFERENCES orders(id) ON DELETE CASCADE,
    seat_id UUID REFERENCES seats(id),
    ticket_code VARCHAR(100) UNIQUE NOT NULL, -- Mã hash dùng làm mã QR soát vé
    check_in_status BOOLEAN DEFAULT FALSE,
    check_in_time TIMESTAMP
);
```

## 4. CHI TIẾT KỸ THUẬT & LUỒNG XỬ LÝ (Technical Design)

### A. Cơ Chế Đặt Vé Tốc Độ Cao Bằng Redis Lua Script
Khi user thực hiện mua vé, luồng xử lý đi qua Redis để đảm bảo tốc độ và an toàn:
1. Mỗi `Concert` khi mở bán sẽ được pre-load sơ đồ ghế trống vào Redis Set: `concert:{concertId}:zone:{zoneId}:available` -> Lưu trữ các `seatId`.
2. Khi user gửi mutation `holdSeat(seatId, userId)`:
   Hệ thống gọi một Lua Script thực thi nguyên tử trên Redis:
   - Check xem `seatId` có trong Set `available` không.
   - Nếu có: Xóa khỏi Set `available`, đồng thời thêm vào Hash `concert:{concertId}:held` -> `{ seatId: userId }` và Redis Key `seat:{seatId}:holder` -> `userId` (set TTL là 10 phút). Trả về `1` (Success).
   - Nếu không có: Trả về `0` (Sold Out hoặc đang bị người khác giữ).
3. Database PostgreSQL chỉ được update trạng thái ghế khi bước trên thành công.

*Mã Lua Script mẫu trên backend:*
```lua
local avail_key = KEYS[1] -- concert:{concertId}:zone:{zoneId}:available
local held_key = KEYS[2]  -- concert:{concertId}:held
local seat_id = ARGV[1]   -- seatId
local user_id = ARGV[2]   -- userId

local is_avail = redis.call('SISMEMBER', avail_key, seat_id)
if is_avail == 1 then
    redis.call('SREM', avail_key, seat_id)
    redis.call('HSET', held_key, seat_id, user_id)
    return 1
else
    return 0
end
```

### B. Giải Phóng Ghế Hết Hạn Bằng RabbitMQ
1. Khi giữ ghế thành công ở bước A, Spring Boot sinh ra Order ở trạng thái `PENDING`.
2. Đồng thời, gửi một message vào Exchange RabbitMQ với thời gian delay là 10 phút.
3. Consumer lắng nghe message này sau 10 phút:
   - Kiểm tra trạng thái đơn hàng trong DB.
   - Nếu vẫn là `PENDING` (chưa thanh toán): Cập nhật trạng thái đơn hàng thành `EXPIRED`.
   - Gửi sự kiện trả ghế về Redis: Lua script chạy ngược lại (Xóa ghế khỏi Hash `held` và trả lại Set `available`).
   - Phát sự kiện qua GraphQL Subscription để client cập nhật giao diện (ghế chuyển lại sang màu xanh lá).

### C. gRPC Ticket Verification (Soát vé B2B)
1. Định nghĩa file protobuf `ticket.proto`:
```protobuf
syntax = "proto3";

package ticket;

option java_multiple_files = true;
option java_package = "com.ticketrush.grpc";

service TicketService {
    rpc VerifyTicket (VerifyRequest) returns (VerifyResponse);
}

message VerifyRequest {
    string ticket_code = 1;
}

message VerifyResponse {
    bool valid = 1;
    string message = 2;
    string seat_number = 3;
    string user_name = 4;
}
```
2. Cổng gRPC Server mở tại port `50051` trên Monolith để các thiết bị soát vé gọi kiểm tra trạng thái vé cực nhanh.

## 5. THIẾT KẾ GIAO DIỆN (UI/UX Mockups & Flow)
- **Home/Concert Detail:** Dark Mode, hình ảnh Banner sắc nét, đồng hồ đếm ngược.
- **Seat Map Visualizer (Next.js SVG):** 
  - Khởi tạo WebSockets connection tới GraphQL Subscription.
  - Khi nhận event `seatStatusChanged(seatId, status)` từ Subscription, Next.js cập nhật trực tiếp fill-color của thẻ `<path>` hoặc `<circle>` tương ứng trên SVG:
    - Trắng/Xám: Ghế trống.
    - Cam: Ghế đang bị giữ (Hold).
    - Đỏ: Ghế của chính mình đang chọn.
    - Đen/Bị gạch: Ghế đã bán.
- **Payment Gateway Interface:** Trang giả lập thanh toán quét mã QR Momo. Nhấn nút "Hoàn thành thanh toán" để gọi Webhook giả lập xác nhận.
